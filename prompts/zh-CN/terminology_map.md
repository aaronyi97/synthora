# Synthora 术语映射表 — 学术用语 → 日常用语

> **用途**：供所有 prompt 文件、后处理层参考。改一处，全局生效。
> **维护规则**：每次发现新的高频学术术语，追加到对应分类。

---

## 一、分析方法类

| 学术/内部用语 | 推荐日常用语 | 备注 |
|------------|------------|------|
| 假设显性化 | 前提假设 / 说清楚你的前提 | |
| 隐含假设挖掘 | 找出没说出来的前提 | |
| 反面论据 / 反面论证 | 反面观点 / 如果情况不同呢？ | |
| 二阶效应 | 连锁反应 / 间接影响 | |
| 认知分歧 | 不同看法 | |
| 分歧标注 | 有不同看法要说出来 | |
| 多源汇系统 | 进出水量的平衡问题（视具体场景替换） | 具体问题具体替换 |
| 成本效益分析框架 | 分析利弊 / 划不划算 | |
| 立场光谱 | 各方立场 / 不同看法 | |
| 前提假设（已是日常用语，保留） | ✅ 可直接使用 | |

## 三、物理/数学专业词（有解释义务）

| 术语 | 加括号解释示例 |
|-----|--------------|
| 托里拆利定律 | 托里拆利定律（水从孔口流出的速度取决于水深） |
| 净现值 | 净现值（把未来的钱折算成今天值多少） |
| 微分方程 | 微分方程（描述变化速率的数学方程） |
| 椭圆积分 | 椭圆积分（一类没有简单公式解的积分） |

## 四、不需要替换的术语（保留原词）

以下术语在目标读者中已有足够认知度，无需替换：

- 人工智能 / AI
- 算法
- 数据
- 模型
- 概率
- 假设（作为普通名词时）

---

## 五、技术题特别说明

`TECHNICAL` / `CODING` / `MATH` 题型中，专业术语是精确性的保障，**不强制替换**。
但仍需在首次出现时加括号解释（如果目标读者可能不熟悉）。
